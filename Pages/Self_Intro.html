Q1. 地学大模型与全栈工程相关

问：“你提到在全栈开源项目里用到了 Docker 和 Hugging Face，还做到了系统线上部署，你写这些代码在科研中有什么价值？”

答题话术：“谢谢老师的提问。传统的 GIS 科研往往止步于发表论文或生成一张静态图表。我独立研发的这套 Vue 3 + FastAPI + Docker 架构体系，其核心价值在于将复杂的地学算法工程化、服务化、可视化。我们在 Hugging Face 上托管 API 容器，意味着可以将地理所先进的地学大模型、时空演化模拟，用极低的成本一键封装成在线交互工具。这不仅能让非地信专业的合作者直观体验地学研究成果，更为未来‘云端协同的地学决策支持系统’搭建好了底层脚手架，实现‘研究成果一键落地运行’。”

Q2. 物理地理与数理分析相关

问：“你参与的虎丘岗遗址黑碳研究，这个纯物理实验课题与你的 GIS 申请方向（技术与应用）有什么联系？”

答题话术：“谢谢老师。这两年的系统物理实验，表面上在洗土样、烧黑碳，实际上锻炼了我两个极关键的能力：第一是地学第一手物理现场数据的采集能力。真正优秀的地信科学家绝不能只在电脑前做‘键盘侠’，必须理解泥土、气候等物理机理；第二是高维时序分析的数理建模能力。我负责处理收集到的磁化率、黑碳含量等核心理化时序。在导师指导下，我引入小波分析（Wavelet Analysis）和小波功率谱去识别地层沉积周期的全新世尺度。这种信号频域分解技术，与卫星遥感时间序列分析、气象时空变化规律挖掘在底层数学上完全互通。这使我未来做 GIS 应用开发时，有极为扎实的物理地学机理支撑。”

Q3. 大数据挖掘与算法设计相关

问：“在‘宝藏小城’大数据的熵权法模型里，为什么要引入人口反比因子？”

答题话术：“谢谢老师。在传统熵权法评选热门旅游地时，北上广深由于基础客流、POI数量极大，会形成绝对统治力，这会导致普通‘宝藏小城’因基数低而被彻底掩盖。为了突出‘反向旅游’中‘小众、高性价比’的独特性，我通过严谨推导，将‘城市常住人口’的倒数作为惩罚约束（反比因子）引入决策公式。这样，在同等旅游热度和社交媒体爆发系数下，小城因基数小，其综合得分反而能实现指数级纠偏。这个因子保证了模型不偏心大都市，真正实现了‘反向挖宝’的数学合理性。”
